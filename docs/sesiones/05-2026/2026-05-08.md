## Trabajo 1 — Nuevo reporte: Usuarios por Dirección Administrativa

**Commits:** `afe5a70`, `1f86dfa`, `24e6318` — `reporte de usuarios por unidades,direcciones, almacenes`

### Problema
No existía un reporte que mostrara, para un almacén (sucursal), sus Direcciones
Administrativas con sus Unidades y los usuarios del sistema asignados a cada unidad.
Se necesitaba para auditar qué funcionarios están registrados por unidad.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Http/Controllers/ReportUsuariosDireccionController.php` | Controlador nuevo: arma la estructura Dirección → Unidades → Usuarios cruzando `sysalmacen.users` con `mamore.people` |
| `resources/views/almacenes/report/aditional/usuariosDireccion/report.blade.php` | Formulario de filtros (selección de almacén) |
| `resources/views/almacenes/report/aditional/usuariosDireccion/list.blade.php` | Tabla de resultados cargada por AJAX |
| `resources/views/almacenes/report/aditional/usuariosDireccion/print.blade.php` | Vista de impresión |
| `routes/web.php` | Rutas `GET /admin/print/almacen-usuarios-direccion` y `POST /admin/print/almacen/usuarios/direccion/list` |
| `database/seeders/MenuItemsTableSeeder.php`, `PermissionsTableSeeder.php`, `DataTypesTableSeeder.php` | Ítem de menú y permisos Voyager para el nuevo reporte |
| `CLAUDE.md` | Documentación del proyecto agregada al repositorio |

### Solución
Se creó el reporte completo (formulario, lista AJAX e impresión) siguiendo el patrón
estándar de reportes del sistema. Las unidades sin usuarios también se muestran con el
texto "Sin usuarios asignados". Los dos commits posteriores del día ajustaron estilos
de la lista/impresión y el registro en menú/permisos.

**Resultado:** Reporte operativo en Reportes → Usuarios por Dirección, con impresión incluida.
