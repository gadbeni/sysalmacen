## Trabajo 2 — Almacenamiento de imágenes en S3 con adaptador resiliente

**Commits:** `45da27e`, `a41cdd3` — `actualizacion en el setting para que pueda subir imagenes al s3` / `actualizacion`

### Problema
Las imágenes (avatares, settings de Voyager) se guardaban solo en disco local del
servidor; se necesitaba almacenarlas en S3. Además, si S3 no respondía, la subida
rompía la página completa.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Filesystem/ResilientAwsS3V3Adapter.php` | Adaptador nuevo de Flysystem: tolera fallos de conexión a S3 sin tumbar la aplicación |
| `app/Providers/AppServiceProvider.php` | Registra el driver `s3-resilient` |
| `config/filesystems.php` | Disco S3 configurado |
| `composer.json` / `composer.lock` | `league/flysystem-aws-s3-v3` agregado |
| `.env.example` | Variables `AWS_*` documentadas |
| `resources/views/vendor/voyager/login.blade.php`, `sidebar.blade.php`, `master.blade.php` | Carga de imágenes desde el nuevo disco |

### Solución
Se integró S3 como disco de almacenamiento para imágenes usando un adaptador
personalizado que captura errores de red: si S3 está caído, la app sigue funcionando
y usa la imagen por defecto en lugar de lanzar excepción.
